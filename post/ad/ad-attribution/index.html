广告投放-归因（二）

做广告投放时，最难回答的一个问题其实不是“花了多少钱”，而是“这笔转化到底是不是广告带来的”。

用户可能先在信息流里看过广告，后来又在短视频里点过一次，最后通过品牌词搜索下单。那这笔订单到底该算给谁？信息流平台会说自己先触达了用户，短视频平台会说用户是点了自己的广告才有后续动作，而搜索渠道也会说最后成交是自己完成的。

这就是广告归因要解决的问题。

简单说，广告归因就是把一次注册、激活、下单、付费等结果，分配给某个广告触点、某个渠道、某个平台或某种投放行为。

如果没有归因，广告主就没法知道：

  • 哪个渠道真的有效
  • 哪种素材带来了更多转化
  • 哪个广告组的成本更低
  • 预算应该继续投给谁，还是及时止损

所以归因本质上不是一个“报表展示问题”，而是广告投放里最核心的决策基础之一。

广告归因到底在解决什么

从业务角度看，归因解决的是一个非常具体的问题：

当结果发生之后，这个结果应该记到谁头上？

这里的“结果”可能是很多种事件：

  • App 安装
  • 首次激活
  • 注册成功
  • 表单提交
  • 下单成交
  • 付费金额

而“记到谁头上”，也有不同粒度：

  • 记给某个媒体
  • 记给某个渠道号
  • 记给某个广告账户
  • 记给某个广告组
  • 记给某个创意

归因的最终作用，是帮助投放系统和业务团队建立一套“结果归属规则”，从而支撑后续的出价优化、预算分配、报表分析和财务结算。

为什么广告归因这么重要

如果没有归因，很多投放动作都没法闭环。

比如：

  • 你知道投放消耗了 10 万，但不知道哪一路流量带来了注册
  • 你知道今天有 200 个订单，但不知道哪些订单是广告拉来的
  • 你知道某个媒体点击率很高，但不知道是不是它真的带来了后续付费

一旦归因做得不准，后面几乎所有判断都会受影响：

  • 好渠道可能被误杀
  • 差渠道可能被误判成有效
  • 出价模型学到错误信号
  • 财务对账和渠道结算出现分歧

所以归因不是一个可有可无的附属能力，而是广告系统的核心基础设施。

什么是广告触点

理解归因，先要理解“触点”。

所谓触点，就是用户和广告发生过的一次可记录接触。最常见的触点有两种：

  • 曝光：用户看到了广告
  • 点击：用户点击了广告

如果一个用户在转化前接触过多次广告，那么这些曝光和点击都会成为候选触点。归因系统要做的，就是在这些候选触点里决定“谁应该拿到这次转化的归属”。

举个例子：

  1. 用户周一在平台 A 看到了广告，但没有点击。
  2. 用户周三在平台 B 点击了广告，浏览了落地页。
  3. 用户周五直接打开 App 完成了注册。

这时就会出现典型的归因问题：

  • 算给平台 A，因为它先做了触达？
  • 算给平台 B，因为它带来了点击？
  • 还是不算给任何平台，因为用户是自己回来注册的？

不同平台、不同业务和不同归因规则，可能得出完全不同的答案。

常见的归因模型

广告归因之所以复杂，很大程度上是因为“怎么算”本身就没有唯一标准。下面是几种最常见的归因模型。

末次触点归因

Last Touch Attribution 是最常见、也最容易理解的一种。

规则很简单：谁离转化最近，就把结果算给谁。

如果继续用前面的例子：

  • 平台 A 先曝光
  • 平台 B 后点击
  • 用户随后注册

那在末次触点归因下，转化通常会记给平台 B。

优点：

  • 逻辑简单
  • 实现成本低
  • 很适合报表统计和快速结算

问题：

  • 容易高估最后一个渠道的价值
  • 容易忽略前面负责种草和触达的渠道

首次触点归因

First Touch Attribution 和末次触点相反，它强调最早接触用户的那个渠道。

规则是：谁最先把用户带进转化链路，就把结果算给谁。

优点：

  • 更适合看“拉新入口”价值
  • 能体现上游触达和种草的作用

问题：

  • 容易低估后续促成成交的渠道
  • 对效果广告场景不一定总是最合理

点击归因

点击归因强调“用户明确表达过兴趣”的触点。

在很多效果广告平台里，点击归因会比曝光归因更被重视，因为点击意味着用户主动发生了交互行为。

常见规则是：

  • 只要发生过点击，就优先把转化记给点击触点
  • 曝光只有在没有点击时，才作为补充归因依据

优点：

  • 信号更强
  • 广告主更容易接受
  • 一般比纯曝光归因更接近真实效果

问题：

  • 仍然不能证明“点击一定带来了转化”
  • 对品牌广告、内容种草类场景不够友好

曝光归因

曝光归因的逻辑是：即使用户没点广告，只要用户看过广告，广告也可能对后续转化产生影响。

这在品牌广告里很常见，因为品牌广告未必要求用户立刻点击，而是希望提升认知、影响决策。

优点：

  • 能覆盖“看过就种草”的场景
  • 对品牌广告更友好

问题：

  • 容易被质疑“碰瓷转化”
  • 曝光量大时，误归因风险也更高

所以在实务里，曝光归因通常争议最大。

多触点归因

多触点归因认为，用户转化往往不是某一个渠道单独完成的，而是多个触点共同作用的结果。

因此它不再问“到底是谁拿走全部归因”，而是改成：

这次转化，应该按什么权重分给多个触点？

常见做法包括：

  • 平均分配
  • 首尾加权
  • 按时间衰减
  • 按模型估算贡献度

优点：

  • 更接近真实用户决策过程
  • 能反映上游触达和下游收口的协同作用

问题：

  • 实现和解释都更复杂
  • 广告主、媒体、平台未必都愿意接受同一套分配规则

什么是归因窗口

归因窗口是归因里非常关键、也经常被忽略的概念。

它解决的是另一个问题：

广告触点发生后，多长时间内出现的转化，还算这次广告带来的？

比如：

  • 点击后 1 天内转化算归因
  • 点击后 7 天内转化算归因
  • 曝光后 24 小时内转化算归因

这就是归因窗口。

窗口越长，平台通常越容易“拿到”更多转化；窗口越短，归因结果通常越保守。

所以不同平台即使看的是同一批用户，也可能因为窗口不同而报出不同结果。

为什么广告归因总会有争议

广告归因会反复出现争议，并不是因为某一方一定在说谎，而是因为这个问题天然就很难有唯一标准答案。

最常见的争议来源有下面几类。

1. 口径不一致

不同平台对“转化”本身的定义就可能不同。

比如：

  • 有的平台把注册成功算转化
  • 有的平台把激活成功算转化
  • 有的平台把提交表单就算转化

如果连转化事件定义都不一样，报表自然没法对齐。

2. 归因窗口不同

平台 A 用点击后 7 天，平台 B 用点击后 1 天，最终结果一定不一样。

窗口不是技术细节，而是直接决定归因数量的核心参数。

3. 多个平台都“有道理”

用户在转化前往往会被多个平台反复触达。

于是就会出现这种情况：

  • 平台 A 说自己先触达了用户
  • 平台 B 说自己带来了点击
  • 平台 C 说最终成交前最后一次访问来自自己

每个平台都能讲出自己的逻辑，所以争议几乎不可避免。

4. 跨设备和跨端丢失

用户可能在手机上看到广告，在电脑上完成下单；也可能在一个 App 被触达，在另一个端完成注册。

只要用户标识打不通，归因链路就会断，最后只能做不完整归因，甚至完全漏掉。

5. 自然量和广告量混在一起

有些用户本来就会来，只是刚好也看到了广告。

这时平台会倾向于说“我覆盖到了这个用户，所以转化和我有关”，而广告主更关心的是：

如果没有这次广告，这个用户还会不会自己来？

这个问题其实是在讨论“增量价值”，而不是单纯的归因归属。

6. 商业激励不一致

广告平台希望证明自己有效，广告主希望成本更真实，代理商和渠道方也会有自己的 KPI。

当各方激励不同的时候，围绕归因口径的博弈就会持续存在。

归因和增量，不是一回事

这是很多人最容易混淆的地方。

归因回答的是：

这次转化应该算给谁？

而增量回答的是：

如果没有这次广告，这个转化还会不会发生？

这两个问题看起来接近，但并不相同。

举个例子：

  • 用户本来就打算买这件商品
  • 广告刚好在下单前又触达了他一次
  • 按末次触点归因，这个订单可能会记给广告

但从增量角度看，这个广告未必真正创造了新增价值。

所以成熟的广告团队通常不会只看归因报表，还会结合增量实验、留存表现和真实收益来做判断。

归因在工程上通常怎么做

从系统实现角度看，归因通常离不开下面几个能力：

1. 事件采集

先采集足够完整的事件：

  • 曝光
  • 点击
  • 安装
  • 激活
  • 注册
  • 付费

如果上游事件缺失，后面就没法归因。

2. 用户标识关联

要尽量把同一个用户在不同时间、不同端上的行为关联起来。

常见标识可能包括：

  • 设备 ID
  • 用户 ID
  • 渠道参数
  • 点击 ID
  • 安装回传参数

3. 归因规则引擎

系统需要一套明确规则，决定：

  • 优先点击还是优先曝光
  • 多个点击冲突时按什么规则选
  • 窗口多长
  • 是否支持多触点分配

4. 转化回传

广告平台只有拿到后链路转化数据，才能真正做效果归因和出价优化。

所以很多系统里，注册、下单、付费等事件都会回传给广告平台，作为优化和报表依据。

实际工作里，怎样减少归因争议

归因争议很难彻底消失，但可以尽量降低。

比较常见的做法有：

  • 先统一转化事件定义
  • 先对齐归因窗口
  • 明确点击和曝光的优先级
  • 明确最终对账以哪套报表为准
  • 区分“平台归因报表”和“内部 BI 口径”
  • 用实验方法去验证真实增量，而不是只看归因量

对于广告主来说，一个更实用的思路是：

不要把归因结果当成绝对真相，而要把它当成一套有假设前提的衡量方法。

结语

广告归因本质上是在回答一件很难但又无法回避的事：

一次业务结果，到底该记给哪个广告触点？

它之所以重要，是因为它直接影响预算分配、投放优化、效果评估和财务结算；它之所以总有争议，是因为用户路径复杂、数据天然不完整、各方口径也未必一致。

如果只记一句话，可以把广告归因理解成：

用一套规则，把结果分配给某个广告来源。

但真正成熟的投放体系，往往不会只停留在“归因归给谁”，而会继续追问：

这个结果到底是不是广告真正创造的增量？

这也是为什么广告归因永远重要，但又永远不可能只有一个标准答案。
